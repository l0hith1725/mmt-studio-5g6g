# TC-REG-003: Attach/Detach Re-attach Cycle

## Overview
This test validates the UE's ability to perform repeated registration and deregistration cycles. It exercises the complete lifecycle: register, verify REGISTERED state, deregister, verify DEREGISTERED state, and repeat. This stress test catches issues like stale NAS contexts, SQN desynchronization, memory leaks, and context cleanup failures that only appear after multiple cycles.

## 3GPP Background
The 5G registration lifecycle involves two key NAS procedures:

**Registration (TS 24.501 Section 5.5.1.2):** The UE sends a Registration Request, undergoes 5G-AKA authentication, NAS Security Mode Command, and receives Registration Accept. The UE transitions to RM-REGISTERED state.

**Deregistration (TS 24.501 Section 5.5.2.2):** The UE initiates deregistration by sending a Deregistration Request (type: switch-off or normal). The AMF responds with Deregistration Accept (for normal type) and releases the UE context via NGAP UE Context Release Command. The UE transitions to RM-DEREGISTERED state.

Each registration cycle requires the AUSF/UDM to generate fresh authentication vectors. The SQN (Sequence Number) increments with each authentication attempt. After deregistration, the AMF deletes the NAS security context, so re-registration requires full authentication.

**Network functions involved:** UE, gNB, AMF, AUSF, UDM
**State transitions per cycle:** DEREGISTERED -> REGISTERED -> DEREGISTERED

## Problem Statement
- What if the AMF does not properly release the UE context after deregistration, leaving stale entries?
- What if the SQN drifts out of sync after multiple auth cycles, causing permanent auth failure?
- What if memory is leaked on each register/deregister cycle, causing eventual resource exhaustion?
- What if the gNB does not free the RAN UE NGAP ID after deregistration, causing ID exhaustion?
- What if NAS security context from a previous registration persists and conflicts with the new one?

## Test Procedure (Step-by-Step)
1. Create gNB from configuration, establish SCTP, complete NG Setup.
2. **Cycle 1-3:** For each cycle:
   a. Register UE: attach to gNB, Registration Request, 5G-AKA, Security Mode, Registration Accept.
   b. Verify UE is in REGISTERED state.
   c. Deregister UE: send Deregistration Request, wait for Deregistration Accept and UE Context Release.
   d. Verify UE is in DEREGISTERED state.
   e. Pause 500ms between cycles to allow AMF context cleanup.
3. All 3 cycles must complete successfully.

## Expected Behavior
- Each registration cycle completes with fresh 5G-AKA authentication.
- SQN increments correctly with each authentication (no AUTS resync needed).
- UE FSM cleanly transitions DEREGISTERED -> REGISTERED -> DEREGISTERED each cycle.
- AMF releases UE context fully between cycles.
- No stale context or security key carryover between cycles.

## Pass/Fail Criteria
- **Pass:** All 3 register/deregister cycles complete successfully; UE reaches REGISTERED then DEREGISTERED in each cycle.
- **Fail:** Any cycle fails to register or deregister; timeout exceeded; SQN sync failure on later cycles.

## Key Concepts for Training

### Registration Management States
The 5G NAS layer defines two Registration Management (RM) states: RM-REGISTERED and RM-DEREGISTERED. In RM-REGISTERED, the UE has an active MM context on the AMF and can access network services. In RM-DEREGISTERED, the UE has no active context and must register again before any service. The transitions between these states are triggered by Registration Accept and Deregistration Accept messages.

### SQN Management Across Cycles
The Sequence Number (SQN) is a 48-bit counter maintained by both the UE (SQN_MS) and the network (SQN_HE in the UDM). Each authentication vector generated by the UDM increments SQN_HE. The UE accepts an AUTN only if the SQN in it is within a certain range of its stored SQN_MS. If SQN_HE runs too far ahead (e.g., due to failed attempts), the UE triggers AUTS resynchronization. Proper SQN management across cycles is essential.

### UE Context Release
After deregistration, the AMF sends an NGAP UE Context Release Command to the gNB. The gNB releases all radio resources, removes the UE context, frees the RAN UE NGAP ID, and responds with UE Context Release Complete. This ensures clean state for the next registration cycle.

## Common Issues and Troubleshooting
| Issue | Symptom | Resolution |
|-------|---------|------------|
| Stale NAS context | Second registration fails with cause "Implicitly de-registered" | Verify AMF releases context on deregistration |
| SQN out of sync | Auth failure with AUTS on cycle 2+ | Check UDM SQN increment logic |
| RAN UE NGAP ID leak | gNB runs out of IDs after many cycles | Verify ID release in UE Context Release |
| Timer expiry | Deregistration times out | Check AMF processing of Deregistration Request |
| Race condition | Register starts before deregister completes | Ensure 500ms pause and state verification |

## Quiz Questions
1. After a UE deregisters, why must it perform full 5G-AKA authentication again when re-registering (instead of reusing the previous security context)?
   *Answer: Upon deregistration, the AMF deletes the UE's NAS security context (KAMF, KNASint, KNASenc). The UE also clears its stored security context. Without a valid security context, the AMF cannot verify NAS integrity, so it must authenticate the UE from scratch.*

2. What is the purpose of the 500ms pause between deregistration and re-registration cycles?
   *Answer: To allow the AMF to fully process the Deregistration Request, release the UE context, send UE Context Release Command to the gNB, and clean up internal state. Without this pause, the new registration might collide with ongoing cleanup, causing race conditions.*

3. If a UE successfully registers 3 times in a row, how many times has the SQN_HE been incremented in the UDM (assuming no resync)?
   *Answer: At least 3 times (once per authentication). Each cycle generates a new authentication vector which increments SQN_HE. If the UDM pre-generates multiple vectors per request, SQN_HE may increment by more than 1 per cycle.*
